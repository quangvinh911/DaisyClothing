# DaisyDaily Project-Scoped Rules for AI Agent

## Core Architecture
* **Frontend**: Next.js 16 (App Router + Turbopack) using route groups (e.g. `client/src/app/(public)/`). Styling is SCSS with BEM conventions.
* **Backend**: NestJS (TypeScript) with Prisma ORM (PostgreSQL).
* **Package Manager**: npm (Workspace config manages both `client` and `server`).

## Fast Startup Commands
* **Run Dev Environment**: Propose running `npm run dev` from the root directory to run both NestJS and Next.js concurrently.
* **Ports**: Next.js dev server runs on port `4200` (`http://localhost:4200`); NestJS runs on port `5000` (`http://localhost:5000`).

## Dev Troubleshooting
* **Mysterious Next.js 404/Build Errors**: If Next.js returns a 404 for `/` or fails to compile due to autogenerated files like `.next/dev/types/routes.d.ts`, kill the active process, delete the `.next` directory (`client/.next`), and restart `npm run dev`.

## Implementation Principles
* **Error Tolerant SSR/SSG**: Ensure public route page components use catch clauses on API fetch methods (e.g. `api.getLatestPosts(3).catch(() => null)`) to leverage fallback data.
* **Prisma**: Before writing migrations or DB models, check existing migrations in `server/prisma/migrations`.
